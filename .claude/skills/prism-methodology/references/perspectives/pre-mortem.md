# Pre-mortem — Prospective Failure Analysis

It is six months from now. This approach has failed and the evaluators scored it poorly. Write the post-mortem. Be brutally specific — what exactly went wrong, when did it start going wrong, what was the first sign we ignored? Tell a concrete story with real technical details, not a list of generic risks. Name the file that had the bug, the API call that timed out, the user flow that confused everyone.
